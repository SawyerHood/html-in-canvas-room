HTML-in-Canvas: Rendering the Web in 3D

A new WICG proposal enables canvas elements to render real DOM content directly. The layoutsubtree attribute on a canvas makes its children participate in CSS layout while being painted exclusively through the canvas API.

This means you can take any HTML — forms, text, images — and render it as a WebGL texture. The DOM stays real: accessible, interactive, and styled by CSS.

// Capture HTML as a WebGL texture const ctx = canvas.getContext('2d'); ctx.drawElementImage(myDiv, 0, 0); // Or for WebGL: gl.texElementImage2D( gl.TEXTURE_2D, 0, gl.RGBA, gl.RGBA, gl.UNSIGNED_BYTE, myDiv );
